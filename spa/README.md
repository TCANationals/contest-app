# TCA Timer — Judge SPA

React + Vite + TypeScript implementation of the judge web app described in
`TCA_Timer_Design_Spec.docx.md` §10.

## Commands

```bash
npm install
npm run lint      # tsc -b --noEmit
npm test          # vitest run
npm run dev       # vite dev server on :5173
npm run build     # tsc -b && vite build
npm run preview   # vite preview (serves ./dist on :4173)
```

## Demo mode

Because the backend is a separate deployment, the SPA ships with a built-in
demo transport so humans can click through every screen without a running
server.

Enable it by either:

- Appending `?demo=1` to any URL — e.g. `http://localhost:5173/?demo=1&room=demo-room`.
- Running dev/build with `VITE_DEMO_MODE=1`.

Demo mode intercepts `fetch` for `/api/judge/*` endpoints and swaps the judge
WebSocket (`/judge?...`) with an in-memory simulator that implements the timer
state machine from §6.5, seeds a synthetic help-call after a few seconds, and
echoes back `PONG` frames so time-sync works.

## Structure

```
src/
  main.tsx                      Mount React + Router + React-Query + CSS
  App.tsx                       Top-level routes (including chrome-less /projector)
  components/
    AppLayout.tsx               Header, sidebar rail (desktop), bottom tabbar (mobile)
    CountdownWithBorder.tsx     Shared digit renderer (§9.2.4 / §10.5)
    ConnectionPill.tsx          WS/DB status badge
  pages/
    Timer.tsx                   §10.4 timer controls + 30s undo
    Help.tsx                    §7 queue, chime on empty→non-empty
    Log.tsx                     §11.2 /api/judge/log w/ CSV export
    Settings.tsx                §7.4.4 SMS/email/quiet-hours preferences
    Rooms.tsx                   §10.2 MRU-first room list
    Projector.tsx               §10.5 full-screen projection view
  hooks/
    useWebSocket.ts             §6.4 WS connect, reconnect w/ jitter, fg resume
    useTimer.ts                 §6.3 OffsetTracker (median-of-6 over window of 8)
  store/
    index.ts                    Zustand app store + send() transport
    types.ts                    TimerState, HelpQueue, frame types
  api/
    client.ts                   Typed fetch wrappers for /api/judge/*
  lib/
    rooms.ts                    localStorage MRU (§10.2)
    quietHours.ts               §7.4 timezone-aware bitmask evaluation
    demoMode.ts                 In-memory simulator for local preview
  pwa/
    manifest.webmanifest        Installable PWA manifest (also generated by VitePWA)
    sw.ts                       Reserved SW entry (plugin generates the real SW)
  index.css                     Tailwind entry + animations (pulse, flash)
```
